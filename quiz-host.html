Játék verzió: 2.5

📋 VERZIÓELŐZMÉNYEK

2.5
Akasztófa játékmódhoz szó adatbázis és indítási felület átdolgozása — Firebase-alapú, perzisztens szó adatbázis a host felületen (bal panel, Szó adatbázis fül). Kategóriák szabadon meghatározhatók, átnevezhetők, törölhetők és 30 szín közül egyedi színnel jelölhetők. Szavakhoz 2 opcionális segítség adható. Tömeges JSON import/export, manuális szerkesztés. Az Akasztófa indítása modál átdolgozva: adatbázisból tölt, kategória grid, 1–50 szó, session-szintű kijátszott-szó nyomkövetés.
2.4
Akasztófa játékmód következetesség javítása — Segítség kérés (💡) gomb: 2 alkalom, mindkettő −1 életet von le. Első segítség szöveges tipp, második egy véletlen betűt fed fel. Solo módban csak az aktuális tippelő kérhet segítséget. Javított pontszámlálás és életszámok szinkronizálása.
2.3
Akasztófa játékmód játék vége menüpont átdolgozása — Külön eredmény képernyő: túlélők és elesettek két oszlopban, személyes kiemelés (megmenekültél / elesettél), statisztika. Solo módban az adott szó is megjelenik. „Vissza a játékba" gomb a host visszatérésekor aktiválódik.
2.2
Akasztófa játékmód intro bevezetése — A játék indításakor ~16 másodperces filmszerű bevezető animáció jelenik meg a játékosoknál (fekete háttér, 3 egymás után megjelenő mondatos sztori). A host felületen visszaszámlálós gyűrű mutatja az intro hátralévő idejét.
2.1
Akasztófa játékmód integrálása — Két játékmód: Kooperatív (mindenki együtt tippel) és Solo kör (minden játékos saját szót kap, felváltva tippelnek). Magyar ábécés billentyűzet, életek, akasztófa SVG grafika, valós idejű Firebase szinkron. Játék végi túlélő/elesett eredmény.
2.0
Csapatverseny játékmód — Játékosok csapatokba oszthatók (2–5 csapat), minden csapatnak egyedi neve adható. A csapatpontok valós időben összesítődnek: az egyes játékosok által szerzett pontok a saját csapatuk összpontszámába kerülnek. A legvégén a legtöbb pontot szerzett csapat győz. Játék közben a játékosnál „Csapattársak" és „Csapat pontunk" jelenik meg. Az eredmény képernyőn csapat dobogó és egyéni hozzájárulás is látható.
1.30
Játékos elérhetőség státusz ellenőrzés + "Vissza a játékba" menüpont létrehozása — soft disconnect (játékos node megmarad képernyőzároláskor), automatikus újracsatlakozás, zöld/piros kapcsolat-jelző a játékos felületen. Eredmény képernyőn új "Vissza a játékba" gomb az azonnali visszatéréshez.
1.29
Host felület bal és jobb oldali átdolgozása + váróterem bugfix — összecsukható bal/jobb panel (◀▶ toggle gomb, animációval), vastagabb és mindig látható kezelősáv. Váróterem kártya 2-soros elrendezés (nincs több vízszintes görgetés).
1.28
Host jobb panel átrendezés — új sorrend: Szavazás → Váróterem → Játékban. A "Pontállás" fejléc eltávolítva. Váróteremben új "Eltávolítás" gomb (narancs) a játékos csendes eltávolítására, mellette a meglévő "Beenged" gomb. A "Játékban" szekció fejlécébe kerültek a játékos-számláló és Nulláz gombok.
1.27
Stale answer fix — válaszok kérdés-azonosítóval (qIndex) vannak ellátva, így elavult válaszok nem keverednek az aktuális kérdéshez. A answers.remove() befejezése előtt az új kérdés nem kerül kiküldésre (szekvenciális Firebase műveletek). Host onDisconnect védelem.
1.26
Jelentés funkció — minden kérdéshez „Jelentés" gomb az Eredményeim képernyőn. Megnyomásával modal nyílik a kérdés adataival, szabad szövegmező a meglátáshoz, és automatikus emailküldés a fejlesztőnek (FormSubmit API + mailto: tartalék).
1.25
Eredményeim hiba javítás — a helyes válasz mostantól minden esetben pontosan jelenik meg, még hálózati késés esetén is (revealedForIndex guard)
1.24
Játékosok látják a kérdések sorszámát (#num) a játék közben és az Eredményeim képernyőn is
1.23
Kérdés sorszámozás — minden kérdéshez egyedi sorszám (#num), host listában és játék közben megjelenik. Import JSON-ban sorszám mező, visszamenőleges sorszámozás gomb.
